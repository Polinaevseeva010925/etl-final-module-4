# Итоговое задание по дисциплине ETL-процессы 4 модуля. Реализация ETL-процесса
# Выполнила Евсеева Полина

В репозитории собраны материалы по выполнению 4 заданий, связанных с загрузкой, обработкой и визуализацией данных в Yandex Cloud.  
В рамках работы использовались YDB, Object Storage, Managed Service for Apache Airflow, Managed Service for Apache Kafka, Yandex Data Processing, Yandex Query и DataLens.

## Структура репозитория

repo/
├── 1_task/
├── 2_task/
├── 3_task/
└── 4_task/

Внутри папок по заданиям могут находиться подпапки:
- scripts/ — скрипты и код;
- dags/ — DAG-файлы для Airflow;
- data/ — исходные данные;
- results/ — результаты выполнения и выходные файлы;

## Содержание по заданиям

### 1_task
Подготовка базы данных в YDB, создание таблицы, загрузка данных и настройка трансфера в Object Storage.

### 2_task
Генерация данных, обработка CSV-файла через PySpark, запуск DAG в Managed Service for Apache Airflow и сохранение результата в Parquet.

### 3_task
Настройка инфраструктуры для потоковой обработки данных, генерация JSON-сообщений, работа с Kafka и сохранение результата обработки в бакет.

### 4_task
Создание подключений в DataLens, формирование датасетов и построение аналитических дашбордов по результатам всех заданий.

## Запуск и использование

Для просмотра материалов достаточно открыть нужную папку с заданием.  
Скрипты можно запускать в соответствии с порядком выполнения, описанным в отчете.  
Результаты работы и скриншоты находятся в соответствующих папках внутри каждого задания.
Некоторые файлы могут быть привязаны к облачным сервисам Yandex Cloud и требуют заранее подготовленной инфраструктуры.  
Если в папке есть скриншоты, они отражают успешное выполнение соответствующего этапа задания.
